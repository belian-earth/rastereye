# RasterEye

A VS Code / Positron extension for quickly previewing local GeoTIFF files directly in the editor. Powered by [deck.gl-raster](https://github.com/developmentseed/deck.gl-raster), [deck.gl](https://deck.gl), and [MapLibre GL JS](https://maplibre.org).

> **Note:** RasterEye is designed for viewing local files generated by your workflows. Remote/cloud-hosted rasters are not currently supported.

## Features

- **Click any `.tif` / `.tiff` file** to open it as an interactive map viewer
- **Base layers** -- Light, Dark, OSM (Voyager), Satellite
- **Transparency slider** -- adjust raster opacity 0-100%
- **Single band rendering** with colormap selection:
  - Viridis, Magma, Inferno, Plasma, Turbo, Cividis
  - Spectral, RdYlBu, RdYlGn (diverging)
  - Hot, Greys, Terrain, Coolwarm
- **3-Band composites** -- assign any band to R, G, B channels for true/false colour views
- **Value range controls** -- min/max contrast stretching with auto 2nd/98th percentile defaults
- **Scale/offset support** -- reads GDAL scale and offset metadata for calibrated display values
- **Band descriptions** -- reads band names from GDALMetadata when available
- **Auto CRS reprojection** -- via deck.gl-raster's adaptive mesh reprojection
- **COG and standard GeoTIFF support** -- works with both Cloud Optimized and regular tiled GeoTIFFs
- **Collapsible control panel** -- click the filename header to collapse/expand

## Quick Start

### Install from .vsix

Download the latest `.vsix` from [Releases](https://github.com/h-a-graham/rastereye/releases) and install via:

**VS Code / Positron:** Extensions > Install from VSIX

### Build from Source

#### With Make (recommended)

```bash
git clone https://github.com/h-a-graham/rastereye.git
cd rastereye
make
```

#### Without Make

```bash
git clone https://github.com/h-a-graham/rastereye.git
cd rastereye
npm install
npm run build
npm run package
```

Both produce a `rastereye-*.vsix` file ready to install.

## Build Commands

| Command | Description |
|---------|-------------|
| `make` | Build and package (default) |
| `make build` | Build extension + webview bundles |
| `make package` | Create `.vsix` package |
| `make watch` | Watch mode (live rebuild on save) |
| `make clean` | Remove build artifacts |
| `make install` | Install npm dependencies |
| `make rebuild` | Clean rebuild from scratch |
| `make help` | Show all available commands |

## How It Works

RasterEye registers as a custom editor for `.tif` / `.tiff` files. When you open a local GeoTIFF:

1. A local HTTP server starts (bound to `127.0.0.1`) to serve the file to the webview
2. The webview loads a MapLibre GL map with a deck.gl overlay
3. `COGLayer` from deck.gl-raster handles tile fetching, CRS detection, and reprojection
4. Tiles are decoded on the main thread (compatible with webview sandbox)
5. A custom CPU render pipeline applies colormaps, band selection, and value scaling per-tile

## Architecture

```
src/
  extension.ts          # Entry point -- starts file server, registers editor
  editorProvider.ts     # Custom editor -- injects viewer HTML into webview
  fileServer.ts         # Local HTTP server with CORS for serving GeoTIFF data
webview/
  main.ts               # MapLibre + deck.gl COGLayer + controls + rendering
  colormaps.ts          # 13 colormaps with interpolation + preview
  viewer.html           # HTML template (shared by webview + standalone browser)
```
